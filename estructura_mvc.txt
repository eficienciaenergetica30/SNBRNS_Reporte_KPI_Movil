══════════════════════════════════════════════════════════════
  RESUMEN DE RESPONSABILIDADES POR CAPA
══════════════════════════════════════════════════════════════

  MODELO  (app/models/)
    • Única capa que habla con la base de datos.
    • hana_model.py contiene get_db_connection() y test_db_connection().
    • Si mañana cambias de SAP HANA a PostgreSQL, sólo tocas esta carpeta.

  CONTROLADOR  (app/controllers/)
    • Recibe la petición HTTP, llama al Modelo y decide qué Vista devolver.
    • Cada Blueprint agrupa rutas relacionadas:
        - main_controller.py  → vistas de la interfaz web
        - api_controller.py   → endpoints REST JSON para el frontend
    • No contiene SQL ni HTML.

  VISTA  (app/views/)
    • Templates Jinja2 que renderizan el HTML.
    • No contiene lógica de negocio; sólo presentación.
    • Los archivos JS en static/js/ son la extensión de la Vista en el cliente.

══════════════════════════════════════════════════════════════
  FLUJO DE UNA PETICIÓN (ejemplo: GET /api/sites)
══════════════════════════════════════════════════════════════

  Browser → app.py (entrypoint)
           → app/__init__.py (create_app registra Blueprints)
           → app/controllers/api_controller.py  (ruta /api/sites)
           → app/models/hana_model.py           (consulta HANA)
           → jsonify(sites)                     (respuesta JSON al browser)
